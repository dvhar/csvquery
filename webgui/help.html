CSVQuery Help

What This Software Does

Run SELECT queries on CSV files, display the results interactively in your browser, and save results to new CSV files. It handles large CSV files efficiently — often larger than spreadsheet programs can manage.

It shows you the first several hundred results with options for viewing specific rows or columns. Click any column header to sort results by that column.

How to Save Files

After running a query, click the Save button. Navigate to your desired location, type a filename, and press Enter. All queries on the page will re-run and save to CSV files. The program auto-detects whether to include a header row, but you can override it:

Always output header:
OH select * from data.csv
Never output header:
NOH select * from data.csv

Specifying a File

Click Open to browse files, or double-click a file to add it to your query.

File paths can be saved with aliases for reuse:

Save an alias:
add pets '/home/user/pets.csv'
Use the alias:
select * from pets
Remove the alias (doesn't delete the file):
drop pets

Type show to see all saved aliases. You can also append file format options after the filepath.

Delimiters & File Headers

Add these codes to the beginning of a query or after the file path:

CodeMeaning
NHNo header row
HFirst row is a header
AHAuto-detect header (default behavior)
SDelimiter: spaces
TDelimiter: tabs (also default for .tsv files)
PDelimiter: pipes |
CDelimiter: colons :

Default delimiter is commas and the program guesses whether a header exists based on whether the first row contains numbers.

Selecting Specific Columns

Columns can be referenced by name or number. Use c1, c2 etc. for numbered columns, or use a plain number if you prefix the query with C.

Add M before the query to make commas optional between selections (requires AS for renaming):

Select columns by name:
select c1, c2 'food', c3, dogs, cats from '/home/user/pets.csv'
Select by number (prefix with C):
C select 1, 2 'food', 3, dogs, cats from 'C:\users\dave\pets.csv'
Optional commas (prefix with M C):
M C select 1 2 as 'food' 3 dogs cats from 'C:\users\dave\pets.csv'

Selecting All Columns

All of the following are equivalent to SELECT *:

select * from data.csv
select from data.csv
data.csv

You can also select all columns plus additional ones:

select dogs, * from data.csv

Complex Expressions

Use + - * / % operators and parentheses. Case expressions are also supported.

Arithmetic:
select birthdate+'3 weeks', c1*c2, c1/c2, c1-c2, c1%2, (c1-23)*(c2+34) from data.csv
Case expressions:
select case when c1*2<10 then dog when c1*10>=10.2 then cat else monkey end from data.csv

Value Aliases

Rename selections or use WITH to create aliases usable throughout the query:

Using WITH:
with age*7 as dogyears, 'Mr.'+name as dogname select dogname, dogyears from data.csv order by dogyears
Rename a selection:
select age*7 dogyears from data.csv order by dogyears

Aggregate functions can be combined with GROUP BY. Without GROUP BY, they aggregate all rows into one result row.

For encrypt, each value gets a unique nonce while the program runs. Between restarts, avoid reusing passwords for different values. Encryption doesn't hide value lengths. decrypt returns empty if ciphertext was tampered with. Omitting the password prompts you interactively.

Math functions like log(0) or asin(100) return empty when not a real number. Add NAN before the query to return nan, inf, or -inf instead.

Distinct & Group By

Use DISTINCT after SELECT for unique rows, or GROUP BY to group and aggregate:

Distinct rows:
select distinct dogtypes, fluffiness from data.csv
Count distinct values:
select count(distinct species) from data.csv
Group by column:
select dogtypes, count(*) from data.csv group by dogtypes

Non-aggregate selections in GROUP BY queries return the first found value.

Limiting Rows

Use TOP n after SELECT or LIMIT n at the end:

select top 100 * from data.csv
select * from data.csv limit 100

WHERE Conditions

Combine conditions with AND, OR, XOR, and parentheses. Dates accept natural formats like 'May 18 1955' or '5/18/1955'. Compare null values with the keyword NULL.

Supported operators:

  • =, !=, <>, >, <, >=, <=
  • IS / IS NOT (for null checks)
  • LIKE (pattern matching with % and _)
  • IN (with list or subquery)
  • BETWEEN

! works as NOT and can prefix operators or entire conditions.

Basic comparison:
select * from data.csv where age < 30
BETWEEN with expression:
select * from data.csv where age between (c3-19)*1.2 and 30
IN with list:
select * from data.csv where c1 in (2,3,5,7,11)
Subquery with IN:
select * from pets.csv where species not in (select species from behaviors.csv where prey != 'humans')
IS NOT NULL:
select * from data.csv where name is not null

Sorting Results

Use ORDER BY at the end of the query. Append DESC for descending order:

select * from data.csv order by age, weight desc
select * from data.csv order by c2*c3

Joining Files

Join any number of files. Default is INNER JOIN; use LEFT JOIN to include unmatched rows from the left table:

Left join:
select p.name, p.species, code.c1 from pets.csv p left join codes.csv code on p.species = code.species order by p.age
Mixed joins:
select p.name, code.c1, fur.flufftype from pets.csv p inner join fur.csv fur on p.fluffiness = fur.fluff and fur.flufftype != 'hairless' left join codes.csv code on p.species = code.species or p.genus = species.genus

Join conditions support multiple comparisons with AND, OR, XOR, and parentheses.
